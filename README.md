# Shake It Off - T 型平衡平台控制系統

這是一個基於 STM32 且使用 MPU6050 (IMU) 作為姿態感測的 T 型連桿機構平衡平台專案。系統採用了**速度型 PID (Velocity Form PID)** 搭配**條件式積分 (Conditional Integration Anti-Windup)**，將平台的傾斜誤差直接轉換為伺服馬達的絕對修正角度。

---

## 🛠 1. 如何編譯與燒錄專案

本專案採用 **Makefile** 進行編譯，並可使用 **STM32CubeMX** 進行底層硬體配置。若您習慣使用指令列工具，請參考以下步驟：

1. **硬體配置 (可選)**：
   若需修改腳位或時脈，請使用 **STM32CubeMX** 開啟專案內的 `.ioc` 檔，修改完成後點擊 `GENERATE CODE` (確保 Project Manager 內的 Toolchain/IDE 設為 `Makefile`)。
2. **編譯韌體**：
   打開終端機 (Terminal)，進入專案根目錄 `shake-it-off/`，輸入以下指令進行編譯：
   ```bash
   make -j4
   ```
3. **韌體燒錄**：
   使用 ST-Link 連接開發板後，輸入以下指令將編譯出的 `.bin` 檔燒錄至硬體：
   ```bash
   st-flash write build/*.bin 0x8000000
   ```
4. **查看 Log 輸出**：
   系統透過 USB CDC (虛擬序列埠) 輸出除錯資訊。將 Type-C/Micro-USB 線接上電腦後，可使用 `screen` 指令監控 (請根據實際掛載的 tty 裝置名稱修改)：
   ```bash
   screen /dev/tty.usbmodem* 115200
   ```
   *(要退出 screen，請按 `Ctrl+A` 然後按 `K`，接著按 `Y` 確認)*

## ⚙️ 2. 控制參數調整指南 (Tuning Guide)

程式中最重要的控制參數位於 `Core/Src/main.c` 頂部的 `PV (Private Variables)` 區段。
請根據您硬體的實際反應與馬達扭力來微調這些參數：

```c
#define KP  0.05f   // P 步進增益 (等同於傳統 PID 的 I)
#define KD  0.01f   // D 步進增益 (等同於傳統 PID 的 P)
#define MAX_MECH_ANGLE  30.0f   // 機構安全極限角度
#define MIN_MECH_ANGLE -30.0f
```

* **系統反應太慢（補償無力）**：平台歪掉後回正的速度很慢，請適度 **加大 `KP`** (例如改為 `0.08f` ~ `0.1f`)。
* **系統來回震盪（Overshoot）**：平台回正時衝過頭來回抖動，請 **加大 `KD`** 以增加阻尼效果，或是稍微 **降低 `KP`**。
* **保護硬體極限**：如果馬達因為極度傾斜而發出異音或卡死，代表機構極限不到 30 度，可以適度調低 `MAX_MECH_ANGLE` (例如改為 `20.0f`)，提早觸發系統的軟體極限保護。

## 📐 3. 數學模型與機電轉換原理

系統並非單純地將 PID 誤差加上 90 度餵給伺服馬達，而是透過完整的三角幾何學與致動器傳動比來精確控制，確保線性與非線性的完美對應。

### 步驟一：定義硬體幾何常數
在 `main.c` 的運算中，我們定義了以下機構硬體參數：
* $a = 2.2$：垂直桿長 (cm)
* $b = 4.5$：樞紐高度 (cm)
* $k = 11.4$：平台半寬 (cm)
* $C = 5.1$：致動器圓周長或導程 (馬達轉一圈 360 度，致動器會伸縮的直線距離)

### 步驟二：從目標傾角 $\theta$ 計算致動器絕對長度
當我們想要讓平台維持在 $\theta$ 角度時，右側與左側致動器掛載點的幾何位移 ($dx, dy$) 為：
* **右側位移**：
  $$ dx_R = -a \sin(\theta) + k \cos(\theta) - k $$
  $$ dy_R = b + a \cos(\theta) + k \sin(\theta) $$
* **左側位移**：
  $$ dx_L = -a \sin(\theta) - k \cos(\theta) + k $$
  $$ dy_L = b + a \cos(\theta) - k \sin(\theta) $$

接著利用畢氏定理求出左右兩側伺服馬達所需驅動的絕對直線長度：
$$ l_R = \sqrt{dx_R^2 + dy_R^2} $$
$$ l_L = \sqrt{dx_L^2 + dy_L^2} $$

### 步驟三：長度轉馬達角度與 90 度基準映射
透過公式 $\alpha = \frac{360 \times l}{C}$ 可將直線長度轉換為馬達的總旋轉角度 $\alpha$。
由於我們使用的是 **0~180 度的標準 RC 伺服馬達**，且機構預設以馬達的 90 度作為平台水平狀態，程式採用了以下映射邏輯：
1. 程式在 `main()` 啟動時會先代入 $\theta = 0^\circ$ 算出結果，並將其記錄為水平基準點 `base_alpha`。
2. 在控制迴圈中，將即時算出的目標 $\alpha$ 減去 `base_alpha` 得到差異角度（$\Delta \alpha$）。
3. 將此差值直接加上伺服馬達中立點的 **90 度**，精準算出送給 PWM 的控制角度。

### 步驟四：條件式積分抗飽和 (Conditional Integration Anti-Windup)
若底座傾斜角度過大，可能會算出小於 0 度或大於 180 度的馬達指令，這會讓馬達頂到物理死角。
為了避免內部控制器的積分項（在這裡是累加的 `target_mech_pitch`）盲目長大造成後續的延遲與嚴重震盪，程式在累加前會進行「**預判**」：
* 如果預判的下一步會讓馬達指令超出 `0~180` 的範圍，系統就會**主動丟棄該次的補償步進量**，讓目標角度維持在上一個安全極限。
* 此機制確保控制指令永遠不會超出伺服馬達的物理執行能力，一旦外力解除，系統能達到零延遲瞬間反向回正。
